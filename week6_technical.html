Object Oriented vs. Functional Programming

Object oriented languages like Ruby are ideal in situations in which the programmer wants to add new features to their code. I think the best way to think about this is to consider Pseudocode. When you Pseudocode, you establish all the things you want your code to do and outline how you will accomplish these things. So, by the time you get to writing the actual code, you have already decided that you have a pre-determined set of operations that your code will perform. As you continue to write your code, you'll primarily be adding new things on which you can perform these operations. You can do this by adding new classes that call existing methods and your other classes (classes you've already written) will remain untouched.

Functional programming languages like Java are ideal in the opposite situation: situations in which the programmer does not know how many operations they want to add to their program. So, the number of things to which they are adding opperations is pre-determined prior to writing the code.
